# Konstantin Docs

Назначение: это внешний слой долговременной памяти Hermes. Встроенная память (`USER.md`/`MEMORY.md`) должна хранить только короткое ядро и ссылку на эту папку; подробные устойчивые знания лежат здесь.

## Как пользоваться этой папкой

Перед ответом или изменениями Hermes должен читать не всё подряд, а только релевантный файл:

| Если задача про... | Читать |
|---|---|
| Кто такой Константин, стиль общения, рабочий контекст | `user-context.md` |
| Hermes, модели, Codex, Ollama, Telegram gateway, cron, пути | `infrastructure.md` |
| Как выполнять повторяемую операцию | `runbooks.md` |
| Планирование многошаговой задачи: что именно будет сделано, в каком порядке, как проверить | `plans/` и `plans/README.md` |
| Как устроена сама эта папка и что куда писать | `README.md` |

## Что сюда писать

Писать только знания, которые:

1. будут полезны через недели/месяцы;
2. уменьшают будущие уточнения;
3. имеют понятное место в структуре;
4. не являются одноразовым логом.

## Что сюда не писать

- Полные логи команд.
- Сырые большие outputs моделей.
- Временные TODO.
- Секреты, OAuth-токены, API keys.
- Историю вида “сегодня сделали X”. Для этого есть `session_search`.

## Формат записей

Каждый файл должен разделять:

- **стабильные факты** — можно использовать без перепроверки;
- **решения и rationale** — почему выбрали именно так;
- **операционные правила** — что делать/не делать;
- **открытые вопросы** — что ещё не доказано или требует проверки.

Если информация является пошаговой процедурой — она должна жить в `runbooks.md` или в Hermes skill, а не в обычном описательном разделе.

## Текущая структура

```text
/home/konstantin/docs/
├── README.md           # индекс и правила ведения этой папки
├── user-context.md     # долговременный контекст о пользователе и рабочих доменах
├── infrastructure.md   # фактическая карта AI/Hermes-инфраструктуры
├── runbooks.md         # проверенные процедуры и чеклисты
└── plans/              # планы работ: что не записано — не считается запланированным
    └── README.md       # правила и шаблон планов
```

## Политика встроенной памяти Hermes

Во встроенной `MEMORY.md` достаточно держать короткую ссылку:

```text
Long-term docs live in /home/konstantin/docs/: README.md, user-context.md, infrastructure.md, runbooks.md, plans/. Read relevant file before changing Hermes/memory/cron/Codex/Ollama setup. Multi-step work needs a plan in plans/.
```

Во встроенной `USER.md` оставить только сжатое ядро о Константине: язык, часовой пояс, роль, стиль ответов, политика curated-памяти.
